TL;DR. "Multi-tab" is two distinct capabilities. One was never lost; the other was.
  • PRESENT A — multi-tab within one session (tab-new/tab-list/tab-select): upstream-native in playwright V2 (browserModel.ts + per-connection Chrome tab group via connectedTabGroup.ts).
  • WAS MISSING B — concurrent sessions sharing one Chrome profile: a 2nd client on the same profile evicted the first. This is the feature being restored.

2. Root cause extension-only

The Chrome extension held a single connection slot and explicitly evicted on every new connect — in packages/extension/src/background.ts:

// before
private _activeGroup: ConnectedTabGroup | undefined;   // one slot
private _activeClientName: string | undefined;

async _connectTab(...) {
  await this._cleanupPromise;
  this._disconnect('Another connection is requested');  // ← evicts existing client
  ...
  this._activeGroup = group;                          // single assignment
}

Why extension-only: the relay already isolates per client. extensionContextFactory.ts calls new CDPRelayServer(...) per browser session — each with its own uuid WebSocket path and its own _extensionConnection. Distinct session ids per identity (7df4b464 ≠ 3ad9fea2) confirm separate relays. So a 2nd client already drives a 2nd connectionRequested → _connectTab in the extension; the only contention is the single _activeGroup. No relay change required. ConnectedTabGroup already encapsulates one connection + one Chrome tab group, keyed by its own _groupId — built for isolation, just never allowed to coexist.

3. The patch (applied) — multi-group, no eviction

// after — background.ts
private _activeGroups = new Set<ConnectedTabGroup>();          // N concurrent clients
private _clientNames = new Map<ConnectedTabGroup, string|undefined>();

async _connectTab(...) {
  await this._cleanupPromise;
  // no eviction — clients coexist, each in its own tab group
  const group = new ConnectedTabGroup(connection, tab);
  group.onclose = () => { this._activeGroups.delete(group); this._clientNames.delete(group); };
  this._activeGroups.add(group); this._clientNames.set(group, clientName);
}
// getConnectionStatus aggregates across groups; _disconnect closes all

Isolation rests on each ConnectedTabGroup's own _groupId / attachedTabs filters, so a tab created by client A cannot be claimed into client B's group.
